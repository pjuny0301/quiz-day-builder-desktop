# UX 법칙 적용 메모

## 목적

이 문서는 현재 화면 설계에 어떤 UX 법칙을 적용했는지와, 아직 남은 보완 계획이 무엇인지 정리한다.

## Fitts의 법칙

### 현재 적용

- 학습 완료 화면의 핵심 행동을 `오답만 학습`, `전체 다시 학습` 두 개의 큰 버튼으로 배치했다.
- 완료 화면 버튼은 전체 폭을 넓게 써서 클릭 거리를 줄이고 맞추기 쉬운 크기로 만들었다.
- 객관식 보기 버튼의 최소 높이를 키워서 오답 선택 스트레스를 줄였다.
- 단답형 제출 버튼도 큰 버튼 크기를 유지한다.

### 이번 반영 이유

- 학습 완료 직후에는 사용자가 가장 자주 누르는 버튼이 재학습 계열이다.
- 작은 버튼 여러 개보다 큰 버튼 두 개가 더 빠르고 실수도 적다.

### 다음 계획

- 작은 화면 높이에서는 제출 버튼을 하단 고정형 액션 바에 붙여 도달 거리를 더 줄인다.
- 관리자 화면의 `...` 메뉴 안 액션은 행 간격을 조금 더 벌려 오클릭을 줄인다.
- Day 카드의 `열기`와 `학습` 버튼도 모바일 폭에서 더 큰 터치 타깃으로 키울 수 있다.

## Gestalt의 법칙

### 현재 적용

- 학습 화면을 `진행 정보`, `문제`, `답변`, `완료 요약`처럼 기능 단위 패널로 분리했다.
- 완료 요약의 수치 카드를 같은 카드군으로 묶어 근접성과 유사성을 강조했다.
- 덱, Day, 카드, 세션이 모두 같은 시각 문법 안에서 역할별로 구획되도록 유지했다.

### 이번 반영 이유

- 사용자는 한 화면에서 무엇을 읽고 무엇을 눌러야 하는지 즉시 알아야 한다.
- 숫자 요약과 행동 버튼을 시각적으로 분리하면 완료 후 다음 행동을 더 빠르게 고를 수 있다.

### 다음 계획

- 덱 상세의 Day 카드와 Day 상세의 카드 매칭 카드에 더 강한 그룹 헤더를 넣어 구획감을 높인다.
- 학습 준비 화면에서도 입력 그룹과 실행 그룹의 시각 차이를 더 키운다.
- 메인 화면의 선택 리스트와 우측 미리보기 사이에 선택 강조 라인을 더 분명히 줄 수 있다.

## Miller의 법칙

### 현재 적용

- 학습 완료 화면의 핵심 선택지를 2개로 제한했다.
- 학습 준비 화면은 `학습 범위`, `문제 유형`, `학습 시작`의 3단 구조만 유지한다.
- 객관식 보기 수는 기본 4개로 제한해 작업 기억 부담을 과도하게 늘리지 않는다.

### 이번 반영 이유

- 완료 순간에 선택지가 많으면 사용자가 생각보다 오래 멈춘다.
- 기억용 앱은 기능 수보다 다음 행동의 단순성이 더 중요하다.

### 다음 계획

- 관리자 화면 상단의 보조 정보량을 더 줄여 한 번에 보이는 우선순위를 더 명확히 한다.
- 카드 편집 화면의 대량 입력 영역은 필요할 때만 강조되도록 접기/펼치기 구조를 검토한다.
- 학습 준비 화면에서도 세부 정보는 기본적으로 숨기고 핵심 선택지만 우선 노출한다.

## Progressive Disclosure

### 현재 적용

- `카드 편집` 화면의 대량 입력 영역은 기본적으로 접혀 있고, 필요할 때만 펼친다.
- `학습 준비` 화면의 세션 상세 정보는 기본적으로 숨기고, `세부 정보 보기`를 눌렀을 때만 확장한다.
- 기본 화면에는 범위 선택, 유형 선택, 학습 시작처럼 가장 중요한 흐름만 우선 배치했다.

### 이번 반영 이유

- 카드 편집의 주작업은 한 장씩 작성하는 것이고, 대량 입력은 보조 작업이다.
- 학습 준비의 주작업은 빨리 결정하는 것이고, 세부 통계 확인은 보조 작업이다.
- 사용자가 처음부터 모든 옵션을 볼 필요가 없는 곳은 과감히 접는 편이 집중과 속도에 유리하다.

### 다음 계획

- 덱 상세의 보조 통계를 접을 수 있는 요약 토글을 검토한다.
- 설정 화면에서도 고급 옵션이 생기면 기본값 영역과 고급 영역을 분리한다.

## F 패턴

### 현재 적용

- 메인 `덱 목록` 화면은 3열 덱 카드 그리드를 위에서 아래로 훑는 스캔이 되도록 정리했다.
- 각 덱 카드도 `이름`, `요약`, `액션`이 예측 가능한 위치에 반복되어 빠른 스캔이 가능하다.
- `학습 준비`와 `카드 편집`도 왼쪽에서 오른쪽으로 갈수록 세부 작업으로 이동하는 구조를 유지한다.

### 이번 반영 이유

- 관리 화면은 사용자가 목록을 빠르게 읽고 선택하는 빈도가 높다.
- 텍스트와 메트릭 위치가 계속 바뀌면 덱이 많아질수록 스캔 비용이 커진다.

### 다음 계획

- 메인 목록에서 최근 사용 덱이나 성적 좋은 덱을 구분하는 시각 표시를 넣어 첫 시선 정착점을 더 명확히 한다.
- Day 상세 화면의 카드 매칭도 위에서 아래로 더 빠르게 읽히도록 카드 헤더 정렬을 더 통일한다.

## Z 패턴

### 현재 적용

- `학습 준비` 화면은 좌상단의 맥락 설명, 우상단의 상단 이동, 좌하단의 설정, 우하단의 `학습 시작`으로 이어지는 흐름을 강화했다.
- 완료 화면도 먼저 요약을 읽고 마지막에 큰 행동 버튼 두 개를 고르게 구성했다.

### 이번 반영 이유

- 시작 화면과 완료 화면은 사용자가 빠르게 의사결정을 내리는 곳이라서 대각선 흐름이 잘 맞는다.
- `무엇을 하는 화면인지`와 `지금 눌러야 하는 것`이 서로 멀리 떨어지지 않아야 한다.

### 다음 계획

- 덱 상세 화면에도 상단 설명, 중간 구조, 하단 행동이라는 삼단 흐름을 더 강하게 넣는다.
- 학습 세션 좌측 진행 요약과 우측 문제 풀이 흐름 간의 시각 연결을 더 선명하게 할 수 있다.

## Aesthetic-Usability Effect

### 현재 적용

- 학습 준비 화면에 별도의 실행 카드와 요약 카드를 두어 단순 폼보다 더 제품답게 보이도록 구성했다.
- 완료 화면의 통계 카드와 큰 재학습 버튼을 시각적으로 정돈해, 기능 이해가 더 쉬워지도록 만들었다.
- 전체적으로 역할별 배경 톤과 카드 형태를 유지해 앱이 일관된 제품처럼 보이게 했다.

### 이번 반영 이유

- 사용자는 보기 좋은 화면에서 같은 기능도 더 이해하기 쉽고 덜 거칠게 느낀다.
- 학습 앱은 반복 사용이 많기 때문에 시각적 피로와 거친 인상을 줄이는 것이 기능만큼 중요하다.

### 다음 계획

- 메인 화면과 덱 상세의 카드 계층감을 한 단계 더 높여 상용 앱 느낌을 더 강화한다.
- 아이콘과 미세한 상태 색상 체계를 정리해 제품 완성도를 더 높인다.

## 3-클릭 법칙

### 현재 적용

- 메인 `덱 목록` 화면에서 자주 쓰는 전체 학습 경로를 `덱 선택 -> 전체 학습 -> 학습 시작` 3단계 안에 넣었다.
- `카드 편집`도 메인 강조 영역에서 바로 진입할 수 있게 두어 깊이를 줄였다.

### 이번 반영 이유

- 학습 앱의 핵심 행동은 빠르게 시작되어야 하고, 너무 깊은 정보 구조는 반복 사용에 불리하다.
- 자주 쓰는 흐름은 세부 탐색보다 얕아야 한다.

### 다음 계획

- Day 단위 빠른 학습도 메인 화면 미리보기 카드에서 더 직접 들어갈 수 있게 검토한다.
- 설정처럼 상대적으로 덜 쓰는 기능은 계속 보조 위치에 두어 깊이와 중요도를 분리한다.

## Pareto 법칙

### 현재 적용

- 메인 강조 영역에서 가장 먼저 노출되는 행동을 `전체 학습`과 `카드 편집` 중심으로 정리했다.
- 완료 화면도 자주 쓰는 상위 행동인 `오답만 학습`, `전체 다시 학습`만 크게 보여준다.
- 카드 편집 화면에서는 핵심 작업인 개별 편집을 앞세우고, 대량 입력은 접어 두었다.

### 이번 반영 이유

- 실제 사용 빈도의 대부분은 학습 시작, 카드 편집, 재학습 같은 소수 행동에 몰린다.
- 자주 쓰지 않는 기능까지 같은 비중으로 보이면 핵심 작업이 느려진다.

### 다음 계획

- 사용 로그가 쌓이면 상위 빈도 행동 기준으로 메인 강조 영역을 더 정교하게 조정할 수 있다.
- 설정과 세부 통계는 핵심 작업을 방해하지 않는 선에서 더 뒤로 물릴 수 있다.

## Jacob의 법칙

### 현재 적용

- 메인 화면은 익숙한 카드 갤러리형 브라우징 구조를 유지한다.
- 설정 화면은 `취소`와 `저장`을 분리한 일반적인 폼 패턴을 따른다.
- 학습 완료 화면은 결과 요약 후 재행동 버튼을 제시하는 익숙한 완료 패턴을 따른다.
- `Ctrl+N`, `Ctrl+S`, 숫자 키 선택처럼 일반적인 데스크톱 단축키 규칙을 사용한다.

### 이번 반영 이유

- 사용자는 새로운 시각 스타일은 받아들일 수 있어도, 기본 상호작용 규칙까지 새로 배워야 하면 피로가 커진다.
- 익숙한 폼, 목록, 단축키 패턴을 쓰면 학습 비용이 줄어든다.

### 다음 계획

- 삭제 동작 앞에 더 익숙한 확인 흐름을 넣을지 검토한다.
- 메인 화면의 현재 강조 덱을 카드 하이라이트로 더 분명하게 만들 수 있다.

## Nielsen 휴리스틱: 기억보다 인식에 의존

### 현재 적용

- 학습 완료 화면은 사용 가능한 다음 행동을 버튼으로 직접 보여주고, 숨은 명령을 외우게 하지 않는다.
- 객관식 보기에는 숫자 힌트를 함께 표시해 사용자가 키 매핑을 기억하지 않아도 바로 인식할 수 있게 했다.
- 저장, 새 덱, 대량 입력 단축키는 버튼 텍스트 옆에 바로 보이게 표시했다.
- 학습 준비 화면은 범위, 문제 수, 유형을 요약 카드에 반복 표시해 사용자가 이전 선택을 머릿속에서 다시 복기하지 않게 했다.

### 이번 반영 이유

- 암기 앱이더라도 UI 사용법까지 같이 암기하게 만들면 불필요한 부담이 생긴다.
- 눈에 보이는 힌트와 반복되는 요약은 사용자의 실수를 줄인다.

### 다음 계획

- 카드 편집 화면에서도 현재 선택된 카드 상태를 더 강하게 표시해 저장 대상을 더 쉽게 인식하도록 만든다.
- Day 상세에서도 카드 수와 최근 기록을 카드 헤더에 더 직접적으로 노출할 수 있다.

## Nielsen 휴리스틱: 유연성과 효율성

### 현재 적용

- 메인 화면에서 `Ctrl+N`으로 새 덱 생성이 가능하다.
- 카드 편집 화면에서 `Ctrl+S`로 저장, `Ctrl+Shift+I`로 대량 입력 토글이 가능하다.
- 객관식은 숫자 키, 단답형은 `Enter`, 완료 화면은 `1`과 `2`로 재학습을 선택할 수 있다.
- 마우스 중심 사용자도 그대로 사용할 수 있고, 숙련 사용자는 키보드로 더 빠르게 움직일 수 있다.

### 이번 반영 이유

- 반복 사용이 많은 학습 앱은 숙련 사용자의 속도 이득이 크다.
- 단축키를 강요하지 않으면서도 제공하면 초보자와 숙련자를 함께 만족시키기 쉽다.

### 다음 계획

- 메인 화면에서 화살표 키로 덱 선택 이동을 지원할 수 있다.
- 학습 세션에서 `Esc`로 학습 준비 화면으로 복귀하는 흐름도 검토할 수 있다.

## Nielsen 휴리스틱: 도움말 제공

### 현재 적용

- 전용 도움말 화면 대신, 각 화면 안에 필요한 힌트만 짧게 배치한다.
- 카드 편집, 학습 준비, 학습 진행, 완료 화면에는 지금 필요한 행동과 단축키를 바로 읽을 수 있는 짧은 안내가 있다.
- 오류나 검증 메시지도 문제 상황이 생긴 바로 그 위치에서 보여준다.

### 이번 반영 이유

- 사용자는 별도 도움말 화면으로 이동하는 것보다, 지금 보고 있는 화면에서 바로 이해되는 힌트를 더 선호한다.
- 학습 앱은 문서를 읽는 순간보다 바로 행동해야 하는 순간의 안내가 더 중요하다.

### 다음 계획

- Day 상세와 덱 상세에도 짧은 문맥 힌트를 더 정교하게 넣을 수 있다.
- 초보자용 첫 실행 힌트는 필요해지면 별도 화면이 아니라 1회성 인라인 가이드로 검토한다.
