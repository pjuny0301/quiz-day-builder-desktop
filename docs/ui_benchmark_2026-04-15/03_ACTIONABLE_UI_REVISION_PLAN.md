# Actionable UI Revision Plan

## 목적

이 문서는 앞으로 현재 앱을 어떤 순서로, 어떤 화면에서, 어떤 텍스트와 스타일을 바꿔가며 개선해야 하는지 매우 구체적으로 적은 실행 문서다.

## 우선순위

- P0: 학습 경험을 참조 앱 수준으로 끌어올리는 수정
- P1: 덱/Day 탐색 밀도를 끌어올리는 수정
- P2: 복습 모드와 보조 기능 확장

## P0. 학습 화면 리디자인

### 대상 화면
- `StudyLauncherWindow.tsx`
- `StudySessionWindow.tsx`
- `styles.css`

### 1. Study Launcher를 리스트형 행동 화면으로 바꾸기

현재 문제:
- 화면이 `폼 패널`처럼 보인다.
- 좌상단 큰 여백이 많다.
- 사용자는 `선택하고 시작`보다 `설정 화면에 들어온 느낌`을 받는다.

수정안:
- 헤더 영역을 없애고 상단은 `이전` 버튼만 남긴다.
- 중앙 패널 폭을 `480px ~ 560px`로 고정한다.
- `문제 유형` 텍스트는 제목이 아니라 섹션 레이블처럼 작게 둔다.
- 각 유형 카드는 아래처럼 바꾼다.
  - 높이: `84px ~ 96px`
  - 제목 크기: `28px`
  - 설명 크기: `16px`
  - 선택 시 배경색: 참조 앱처럼 청록 또는 현재 블루 계열 단색 강조
  - 비선택 상태: 흰색 배경 + 얇은 outline
- 버튼 문구는 다음으로 바꾼다.
  - `단답형` 설명: `직접 입력`
  - `선택형` 설명: `보기 선택`
  - `혼합형` 설명: `자동 혼합`
- `학습 시작` 버튼은 패널 하단 full-width로 배치한다.
- 버튼 안의 `학습 시작` 문구는 유지하되, 패널 내 다른 설명 문장은 더 넣지 않는다.

없앨 텍스트:
- 화면 상단의 빈 제목 영역 전체
- 현재 폼 같은 인상만 주는 상단 여백

### 2. Study Session에서 거대한 제목 헤더 제거

현재 문제:
- [02-short-answer-idle.png](study_capture/02-short-answer-idle.png) 기준으로 덱 제목이 지나치게 커서 문제보다 먼저 보인다.
- 학습 화면이 `콘텐츠 소비`가 아니라 `상세 페이지`처럼 보인다.

수정안:
- `WindowShell`의 기본 헤더를 학습 화면에서는 쓰지 않는다.
- 학습 화면 전용 얇은 top bar를 만든다.
- 상단 바 구성:
  - 좌측: `이전` 아이콘 버튼
  - 중앙: `DAY 001` 또는 `전체 카드`
  - 우측: `1/4` 진행 수치, 추후 설정 버튼 자리
- 덱 이름 `Study Comparison Sample`처럼 크게 보이는 H1은 완전히 제거한다.
- 학습 화면 상단 바 높이는 `56px ~ 64px` 수준으로 제한한다.

없앨 텍스트:
- 현재 큰 덱 제목 전체
- `학습 진행` eyebrow

### 3. 사이드바를 없애고 문제 중심 단일 컬럼으로 바꾸기

현재 문제:
- 진행도, 범위, 정오답 비율, 단축키가 좌측 카드로 몰려 있어 문제 집중을 방해한다.
- 참조 앱 대비 시선이 좌측으로 분산된다.

수정안:
- `session-shell`을 `1열` 레이아웃으로 바꾼다.
- 현재 좌측 사이드바 내용은 다음처럼 재배치한다.
  - 상단 바: `DAY / 진행 수치`
  - 문제 카드 바로 아래 얇은 bar: 정오답 비율
  - 완료 화면: 통계 카드로 재배치
- `단축키` 박스는 학습 중 화면에서 제거하고, 필요하면 개발자 모드 또는 tooltip으로만 남긴다.

없앨 텍스트:
- 학습 중 `단축키` 라벨과 설명 박스 전체
- `전체 카드`, `혼합형` 같은 설명 카드형 표시

### 4. 문제 카드 시각 구조 변경

현재 문제:
- 문제 카드가 흰색 카드에 작게 들어가 있다.
- 참조 앱처럼 문제 자체가 화면의 중심이 되지 못한다.

수정안:
- 문제 카드 배경을 하나의 강한 색면으로 바꾼다.
  - 권장: 현재 블루 계열 또는 청록 계열 단색
- 질문 텍스트는 중앙 정렬
- 글자 크기: `clamp(42px, 7vw, 72px)`
- 문제 카드 최소 높이: `34vh`
- 문제 카드 내부 여백: `32px ~ 48px`

추가 규칙:
- 문제가 짧으면 중앙 정렬
- 문제에 이미지가 있으면 이미지 중심 배치, 텍스트는 하단 보조

### 5. 선택형 보기를 세로 스택으로 변경

현재 문제:
- 현재 2열 선택지는 데스크톱에선 공간 효율은 좋지만, 퀴즈 앱 느낌은 약하다.
- 참조 앱의 강점인 `한 줄씩 빠르게 스캔`하는 리듬이 줄어든다.

수정안:
- 보기 버튼은 기본 `1열 세로 스택`으로 바꾼다.
- 각 버튼 높이: 최소 `88px`
- 번호 배지는 좌측 원형 유지
- 텍스트는 좌측 정렬
- 버튼 간 간격: `14px ~ 16px`

색 규칙:
- 기본: 흰색 배경
- 사용자가 고른 오답: 연회색 배경
- 정답: 옅은 빨강 배경 + 빨강 border

### 6. 오답 정답 오버레이 중앙 정렬 보정

현재 문제:
- [04-short-answer-wrong-hold.png](study_capture/04-short-answer-wrong-hold.png) 기준으로 큰 정답 텍스트가 우측으로 밀려 보인다.

수정안:
- `feedback-overlay__wrong`를 `width: 100%`, `display: flex`, `justify-content: center`, `align-items: center`로 바꾼다.
- 정답 텍스트 max-width를 `60%` 또는 `min(680px, 70vw)`로 제한한다.
- 한 글자 또는 짧은 정답일 때도 정확히 중앙에 오도록 `text-align: center`와 `margin: 0 auto`를 강제한다.
- 긴 정답은 `font-size: clamp(44px, 8vw, 96px)`로 조절한다.

### 7. 완료 화면에 오답 목록 Drawer 추가

비교 근거:
- 참조 결과 화면 `Image #1`
- 현재 완료 화면 [09-complete-with-wrong.png](study_capture/09-complete-with-wrong.png)

수정안:
- 현재 `오답만 학습`, `전체 다시 학습` 두 버튼은 유지한다.
- 그 아래 `틀린 문제 보기` Accordion 또는 Drawer를 추가한다.
- 기본 상태는 접힘으로 둔다.
- 펼치면 다음 내용을 보여준다.
  - 문제
  - 정답
  - 사용자가 틀린 유형
- 버튼 문구는 다음처럼 바꾼다.
  - `오답만 학습 (n)` 유지
  - `전체 다시 학습` 유지
  - 추가 버튼: `틀린 문제 보기`

따라가면 안 되는 점:
- 참조 앱처럼 완료 직후 오답 목록을 기본 노출해서 첫 행동을 아래로 밀어내면 안 된다.

## P1. 덱/Day 탐색 화면 개선

### 1. Deck Manager 헤더 압축

현재 문제:
- [manager.png](current_app/manager.png) 기준으로 상단 설명 영역이 너무 크다.
- 실제 덱보다 헤더가 더 많은 공간을 먹는다.

수정안:
- `덱 목록` 텍스트를 `내 덱`으로 바꾼다.
- 제목 크기: 현재보다 축소해서 `40px ~ 48px`
- 아래 설명 문구는 삭제한다.
  - 삭제 대상: `덱 카드를 일반 클릭하면 바로 Day 화면으로 이동합니다...`
- `자동 저장 준비`는 헤더 우측의 작은 상태 텍스트로만 남긴다.
- `새 덱` 버튼은 유지하되, 버튼 내부의 `Ctrl+N` 텍스트는 제거하고 tooltip으로 옮긴다.

### 2. Deck 카드 밀도 증가

수정안:
- 1440px 기준 3열, 1920px 기준 4열까지 허용한다.
- 카드 높이를 줄이고, 카드 이미지 비율은 `4:3`으로 고정한다.
- 카드 제목은 `22px` bold
- 요약 문장은 1줄만 노출하고 ellipsis 처리한다.
- 카드 우상단 더보기는 유지한다.

### 3. Deck Detail Day 리스트를 더 촘촘하게 바꾸기

현재 문제:
- Day 행이 카드형이라 높이가 크다.
- `열기`와 `학습`이 있지만 즉시 스캔성이 약하다.

수정안:
- Day 항목을 `리스트 행`으로 더 납작하게 만든다.
- 행 구성:
  - 좌측: `DAY 001`
  - 중앙: `총 30문제 중 18문제 맞춤`
  - 우측: 정답률 막대 + `열기` + `학습`
- `마지막 3/10` 같은 텍스트는 `최근 3/10` 대신 `최근 3/10` 또는 `최근 점수 3/10` 중 하나로 단일화한다.
- 검색 placeholder는 `예: 1, 12, 103` 대신 `Day 번호 검색`으로 바꾼다.

### 4. Day Detail에 복습 모드 진입 버튼 추가 검토

수정안:
- 현재 `학습` 버튼 외에 추후 `복습` 버튼을 추가할 수 있도록 버튼 슬롯을 남긴다.
- `학습`은 시험 모드, `복습`은 카드 넘김 모드로 분리한다.

## P1. 카드 편집/대량 입력 개선

### 1. 카드 편집 화면 축 재배치

현재 문제:
- 사용자가 원한 구조는 `문제 작성 + 문제 미리보기`가 한 행, `정답 작성 + 정답 미리보기`가 한 행이다.
- 현재 구조는 `문제/정답 작성`과 `문제/정답 미리보기`가 분리돼 있어 의도와 다르다.

수정안:
- 레이아웃을 다음처럼 바꾼다.
  - 좌측: 카드 목록
  - 우측 상단: 문제 작성 | 문제 미리보기
  - 우측 하단: 정답 작성 | 정답 미리보기
- 작은 창에서도 이 `2행 2열` 대응 구조를 유지한다.
- 작아졌을 때는 각 pane 내부 스크롤을 허용하고, 미리보기를 아래로 떨어뜨리지 않는다.

텍스트 변경:
- `문제 미리보기` 유지
- `정답 미리보기` 유지
- 설명 문구는 최소화
  - 삭제 대상: `문제와 분리해 저장해야 학습 모드에서 정확히 활용할 수 있습니다.`

### 2. Bulk Import 에디터 시각 보정

수정안:
- 구분선은 블로그 편집기처럼 보이게 만든다.
  - 두께: `1px`
  - 색: `rgba(150,160,175,0.45)`
  - 위아래 간격: `24px`
- 구분선 앞뒤에 너무 많은 배경 박스를 두지 않는다.
- 상단 안내 문구는 한 줄로 줄인다.
  - 현재 긴 설명을 줄여 `스페이스 두 번 + Enter 또는 Ctrl+Enter로 구분선` 정도로 축약

## 결론

가장 먼저 손봐야 할 것은 `학습 진행 화면`이다. 현재 앱은 기능은 충분하지만, 화면이 공부를 돕는 구조가 아니라 정보를 많이 보여주는 구조에 가깝다. 참조 앱이 강한 지점은 바로 그 반대다. 따라서 `제작 화면은 유지`, `학습 화면은 과감히 단순화`, `결과 화면은 단순 행동 + 접이식 오답 복기`가 가장 맞는 수정 방향이다.
