# Current App Vs Reference

## 목적

이 문서는 현재 `quiz_day_builder_desktop` 앱과 참조 안드로이드 퀴즈앱을 같은 축에서 비교하고, 현재 앱이 우세한 디자인과 반드시 참조해야 하는 디자인을 구분하기 위한 문서다.

## 비교 기준

- 학습 화면 집중도
- 모드 선택의 직관성
- 덱/Day 탐색 속도
- 카드/콘텐츠 제작 효율
- 화면 역할 분리 정도
- 시각 밀도와 가독성
- 다음 행동의 명확성
- 결과 화면에서의 오답 복기 효율

## 현재 앱이 우월한 디자인

### 1. 제작과 소비를 역할별 화면으로 분리한 점

현재 앱은 다음을 별도 화면으로 나눴다.
- 덱 생성
- 덱 설정
- 카드 편집
- 대량 입력
- Day 상세
- 문제 유형 선택
- 학습 진행

참조 앱은 사용 시나리오가 빠르지만, 제작/관리 쪽은 비교 대상 캡처가 부족하다. 현재 앱은 `만드는 앱`으로서 구조가 더 분명하다.

### 2. 학습 완료 후 다음 행동이 명확한 점

현재 앱 완료 화면은 다음 두 행동만 남긴다.
- 오답만 학습
- 전체 다시 학습

이는 암기 앱의 반복 행동과 직접 연결된다. 사용자는 생각하지 않고 다음 행동을 고를 수 있다.

### 3. 정답 피드백의 단순성

현재 앱은 정답 시 큰 `O`, 오답 시 큰 정답만 띄운다.
- 군더더기 문구가 없다.
- 사용자가 결과를 즉시 인지한다.
- Enter로 바로 다음 문제로 넘어갈 수 있다.

이 부분은 참조 앱의 `원형 강조`, `X`, `정답 보기 강조`보다 더 단순하고 빠르다.

### 4. 대량 입력과 단일 카드 편집이 분리된 점

현재 앱은 대량 입력을 별도 에디터로 분리했고, 이미지 붙여넣기까지 지원한다. 이건 참조 앱이 제공하는 소비자용 학습 경험과는 다른 강점이다. 이 강점은 유지해야 한다.

### 5. 결과 화면의 행동 단순성

사용자 제공 참조 결과 화면은 다음이 강하다.
- 오답 목록을 바로 보여준다.
- `틀린 단어 공부하기`, `틀린 단어 다시 풀기`로 즉시 이어진다.

반대로 현재 앱은 다음이 강하다.
- 완료 직후 시선이 흔들리지 않는다.
- 행동이 `오답만 학습`, `전체 다시 학습` 두 개로 정리되어 있다.
- 결과 화면이 짧고 빠르다.

즉, 완료 화면은 현재 앱이 `행동 단순성`에서 우세하다.

## 참조 앱이 우월한 디자인

### 1. 학습 화면 집중도

참조 앱은 퀴즈 화면에서 다음만 강조한다.
- 얇은 상단 바
- 중앙 문제
- 큰 보기 버튼
- 선택 결과

현재 앱은 학습 화면에서도 다음이 너무 크다.
- 헤더 영역
- 덱 제목
- 왼쪽 진행 사이드바
- 정보 카드형 패널

결과적으로 현재 앱은 `공부하는 화면`보다 `관리 화면`에 가깝게 보인다.

### 2. 라이브러리의 시각 밀도

참조 앱은 한 화면에서 많은 덱을 빠르게 훑을 수 있다. 현재 앱 관리 화면은 여백이 많고 실제로 보이는 덱 수가 적다. 덱이 늘어날수록 탐색 속도에서 손해를 본다.

### 3. Day 진입 구조

참조 앱은 Day 한 줄에 `List`, `Quiz`, `Play`를 바로 둔다. 사용자는 Day를 보는 순간 행동을 결정할 수 있다. 현재 앱도 `열기`, `학습`이 있지만, 행의 정보 밀도와 행동 배치가 덜 날렵하다.

### 4. 학습 모드 선택의 즉시성

현재 앱의 `Study Launcher`는 이미 좋아졌지만, 여전히 하나의 큰 폼 패널 느낌이 강하다. 참조 앱은 `퀴즈 선택 화면` 자체가 곧 행동 목록처럼 보인다. 이 차이가 있다.

### 5. 카드/리뷰 모드 분리

참조 앱은 `Quiz`와 `Play`를 확실히 나눈다. 현재 앱은 `Day Detail`이 읽기용이긴 하지만, 카드/슬라이드 소비 모드까지는 아직 아니다.

### 6. 결과 화면의 오답 복기 밀도

사용자 제공 참조 결과 화면은 완료 직후 곧바로 오답 목록을 읽을 수 있다. 현재 앱은 완료 행동은 좋지만, 어떤 문제를 틀렸는지 바로 보이지 않는다. 그래서 `행동은 현재 앱`, `복기 정보는 참조 앱` 쪽이 각각 강하다.

## 반드시 따라가야 하는 디자인

### 따라가야 하는 것

- 학습 화면 상단을 얇게 만들기
- 문제 텍스트를 화면 중심으로 끌어오기
- 보기 버튼을 더 단순하고 더 크게 만들기
- 라이브러리 그리드에 더 많은 덱을 보이게 만들기
- Day 목록 한 줄의 행동성을 높이기
- 학습 설정을 별도 작은 설정 패널 또는 Drawer로 분리하기
- `Quiz`와 `Play`에 해당하는 `시험 모드`와 `복습 모드`를 분리하기
- 완료 화면 아래에 오답 목록을 펼칠 수 있게 만들기

### 따라가면 안 되는 것

- 현재 앱의 대량 입력 구조를 희생하면서 모바일 UI를 흉내 내는 것
- 설정을 학습 화면 안으로 다시 집어넣는 것
- 덱 편집 기능을 카드면 위로 다시 끌어올리는 것
- 제작자 도구와 학습자 화면의 역할을 섞는 것
- 완료 화면을 오답 리스트로 과도하게 길게 만들어 핵심 행동을 밀어내는 것

## 최종 판단

현재 앱은 `관리/제작 툴`로서는 이미 참조 앱보다 낫다. 반대로 `학습/소비 화면`, `라이브러리 브라우징`, `결과 후 오답 복기 화면`은 참조 앱이 더 낫다. 따라서 방향은 하나다.

- 제작 화면은 현재 앱의 강점을 유지한다.
- 학습 화면은 참조 앱의 시각 구조를 적극적으로 가져온다.
- 라이브러리와 Day 목록은 참조 앱처럼 더 촘촘하게 만든다.
- 완료 화면은 현재 앱의 단순 행동 구조를 유지하되, 참조 앱처럼 오답 목록을 `접을 수 있는 하단 영역`으로 추가한다.
- 단일 메인 창 구조와 이름 기반 자동화는 계속 유지한다.
