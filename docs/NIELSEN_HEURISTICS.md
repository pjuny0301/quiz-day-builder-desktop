# Nielsen 10 휴리스틱 체크리스트

## 목적

이 문서는 현재 앱이 Nielsen 10 휴리스틱을 어떻게 반영하고 있는지 점검하는 체크리스트다.  
각 항목은 `현재 적용`, `이번 반영 이유`, `다음 계획`으로 나눠 관리한다.

## 1. 시스템 상태를 사용자에게 보여줄 것

### 현재 적용

- 모든 주요 화면 상단에 자동 저장 상태 문구를 표시한다.
- 로딩 중 화면과 불러오기 실패 화면을 분리해 보여준다.
- 학습 진행 화면은 진행도, 정답 수, 오답 수를 실시간으로 보여준다.

### 이번 반영 이유

- 사용자가 저장 여부와 현재 진행 상태를 추측하게 두면 불안정하게 느껴진다.

### 다음 계획

- 저장 실패 시 재시도 안내를 더 명확히 보여줄 수 있다.

## 2. 실제 세계와 일치하는 언어 사용

### 현재 적용

- 사용자 노출 문구는 대부분 한국어로 통일했다.
- `덱`, `Day`, `학습`, `정답`, `오답`처럼 학습 맥락에서 바로 이해되는 단어를 사용한다.
- 오류 및 검증 메시지도 기술 용어보다 행동 중심 문장으로 보여준다.

### 이번 반영 이유

- 사용자는 개발자 용어보다 학습 상황에 맞는 말로 더 빠르게 이해한다.

### 다음 계획

- 남은 영어 혼합 표현이 있으면 더 줄일 수 있다.

## 3. 사용자가 제어권을 가질 것

### 현재 적용

- 각 화면에 `뒤로`, `목록으로`, `취소` 같은 복귀 경로를 유지한다.
- 학습 완료 후 사용자가 `오답만 학습`과 `전체 다시 학습` 중 다음 행동을 직접 고른다.

### 이번 반영 이유

- 사용자가 강제로 흐름을 따라가는 느낌을 받지 않아야 한다.

### 다음 계획

- 일부 화면에서 `Esc` 같은 복귀 단축키를 검토할 수 있다.

## 4. 일관성 유지

### 현재 적용

- 모든 화면이 같은 `WindowShell` 구조를 사용한다.
- 버튼 계층은 `Primary`, `Secondary`, `Ghost`, `Danger`로 반복된다.
- 저장 상태, 메트릭 카드, 요약 카드의 시각 문법을 통일했다.

### 이번 반영 이유

- 화면마다 규칙이 바뀌면 학습 비용이 커진다.

### 다음 계획

- Day 카드와 카드 매칭 카드의 세부 정렬 규칙도 더 통일할 수 있다.

## 5. 오류 예방

### 현재 적용

- 덱 이름이 비어 있으면 생성하지 않는다.
- 문제와 정답이 모두 없으면 카드를 저장하지 않는다.
- 덱 삭제와 카드 삭제는 확인 메시지를 먼저 띄운다.
- 대량 입력 형식이 맞지 않으면 가져오지 않는다.

### 이번 반영 이유

- 사후 오류 메시지보다 사전 차단이 더 효율적이다.

### 다음 계획

- 설정 화면의 입력값 오류도 더 직접적인 검증 문구로 보완할 수 있다.

## 6. 기억보다 인식에 의존

### 현재 적용

- 버튼 옆에 단축키 힌트를 함께 표시한다.
- 객관식 보기에는 숫자 힌트를 함께 붙인다.
- 학습 준비 화면은 선택 결과를 오른쪽 요약 카드에 반복 표시한다.

### 이번 반영 이유

- 사용자가 조작 규칙까지 외워야 하면 UI 부담이 커진다.

### 다음 계획

- 현재 선택된 카드나 Day를 더 강하게 표시해 인식 부담을 더 줄일 수 있다.

## 7. 유연성과 효율성

### 현재 적용

- `Ctrl+N`, `Ctrl+S`, `Ctrl+Shift+I`, 숫자 키 선택, `Enter` 제출을 지원한다.
- 같은 기능을 마우스와 키보드 둘 다로 수행할 수 있다.

### 이번 반영 이유

- 초보자는 클릭 기반 흐름을, 숙련 사용자는 단축키 기반 흐름을 원한다.

### 다음 계획

- 화살표 키 이동, `Esc` 복귀 같은 고급 단축키를 추가 검토한다.

## 8. 미니멀한 디자인

### 현재 적용

- 대량 입력과 세션 상세 정보는 기본적으로 접혀 있다.
- 학습 완료 화면의 행동 선택지는 2개만 크게 남겼다.
- 메인 화면에서 핵심 작업을 중심으로 정리하고 세부 기능은 뒤로 밀었다.

### 이번 반영 이유

- 암기 앱은 기능 수보다 다음 행동이 바로 보이는 것이 중요하다.

### 다음 계획

- 메인 화면 보조 정보량을 더 줄여도 되는지 추가 점검할 수 있다.

## 9. 오류 메시지가 명확할 것

### 현재 적용

- `덱 이름을 먼저 입력하세요`, `문제를 먼저 작성하세요`, `정답을 먼저 작성하세요`처럼 행동 지시형 문장을 쓴다.
- 대량 입력 오류도 구분자 규칙을 함께 알려준다.
- 불러오기/저장 실패는 상단 상태 문구와 오류 화면에서 그대로 보여준다.

### 이번 반영 이유

- 오류 상황에서 무엇을 고쳐야 하는지 바로 보여주는 편이 회복이 빠르다.

### 다음 계획

- 저장 실패 시 해결 경로를 한 줄 더 붙일 수 있다.

## 10. 도움말 제공

### 현재 적용

- 별도 도움말 화면은 두지 않고, 각 화면에서 필요한 짧은 힌트를 바로 보여준다.
- 단축키, 자동 저장, 진행 방식, 검증 메시지를 맥락 안에서 안내한다.

### 이번 반영 이유

- 사용자는 별도 도움말로 이동하는 것보다 현재 화면 안의 짧은 안내를 선호한다.

### 다음 계획

- 첫 실행 사용자에게만 한 번 보이는 인라인 가이드를 필요시 검토한다.
