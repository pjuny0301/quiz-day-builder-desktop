# 사용 시나리오

## 목적

이 문서는 현재 앱에서 사용자가 실제로 밟게 되는 주요 시나리오를 정리한다.  
기능 추가나 UI 변경은 아래 시나리오를 깨지 않는 방향으로 진행해야 한다.

## UC-01 덱 생성

- 사용자 목표: 새 학습 덱을 만든다.
- 시작 조건: 메인 `덱 목록` 화면에 있다.
- 기본 흐름:
  1. 사용자가 `새 덱` 버튼을 누른다.
  2. 앱이 `덱 이름 정하기` 화면으로 이동한다.
  3. 사용자가 덱 이름을 입력하고 `덱 만들기`를 누른다.
  4. 앱이 덱을 생성하고 메인 목록으로 돌아간다.
  5. 새 덱이 선택된 상태로 보인다.
  6. 새 덱에는 비어 있는 `Day 1`이 기본으로 생성되어 있다.
- 예외 흐름:
  - 이름이 비어 있으면 생성하지 않는다.

## UC-02 덱 설정 변경

- 사용자 목표: 덱 이름, 지연시간, Day당 카드 수를 바꾼다.
- 시작 조건: 덱이 하나 이상 존재한다.
- 기본 흐름:
  1. 사용자가 메인 목록에서 덱의 `...` 버튼을 누른다.
  2. 앱이 `덱 설정` 화면으로 이동한다.
  3. 사용자가 이름, 지연시간, Day당 카드 수를 수정한다.
  4. 사용자가 `저장`을 누른다.
  5. 앱이 자동 분할 결과를 반영하고 메인 목록으로 돌아간다.

## UC-03 카드 개별 편집

- 사용자 목표: 한 장씩 문제와 정답을 작성하거나 수정한다.
- 시작 조건: 대상 덱이 존재한다.
- 기본 흐름:
  1. 사용자가 덱 상세 또는 관련 진입점에서 `카드 편집` 화면으로 이동한다.
  2. 앱이 첫 카드를 자동 선택하거나 빈 초안을 보여준다.
  3. 사용자가 문제와 정답을 각각 입력한다.
  4. 사용자가 `카드 저장` 또는 `카드 수정 저장`을 누른다.
  5. 앱이 카드와 Day 분할을 자동 저장한다.

## UC-04 카드 대량 입력

- 사용자 목표: 여러 카드를 한 번에 추가한다.
- 시작 조건: `카드 편집` 화면에 있다.
- 기본 흐름:
  1. 사용자가 대량 입력 영역에 여러 줄의 문제/정답 쌍을 붙여넣는다.
  2. 사용자가 `카드 가져오기`를 누른다.
  3. 앱이 유효한 줄만 카드로 변환해 추가한다.
  4. 앱이 Day 구조를 다시 계산한다.

## UC-05 덱 구조 탐색

- 사용자 목표: 덱과 Day 구성을 살펴본다.
- 시작 조건: 카드가 들어 있는 덱이 존재한다.
- 기본 흐름:
  1. 사용자가 메인 그리드에서 덱 카드를 일반 클릭한다.
  2. 앱이 `덱 상세` 화면으로 이동한다.
  3. 앱이 Day 리스트와 최근 기록을 보여준다.
  4. 사용자가 원하는 Day를 열어 `Day 상세` 화면으로 이동한다.
  5. 앱이 그 Day의 문제/정답 매칭을 카드 단위로 보여준다.

## UC-06 전체 범위 학습 시작

- 사용자 목표: 덱 전체 또는 선택한 Day 범위를 학습한다.
- 시작 조건: 카드가 들어 있는 덱이 존재한다.
- 기본 흐름:
  1. 사용자가 `학습 준비` 화면으로 이동한다.
  2. 사용자가 학습 범위를 `전체 카드` 또는 특정 Day로 고른다.
  3. 사용자가 문제 유형을 고른다.
  4. 사용자가 `학습 시작`을 누른다.
  5. 앱이 `학습 진행` 화면에서 한 문제씩 출제한다.

## UC-07 문제 풀이와 자동 진행

- 사용자 목표: 현재 문제를 풀고 바로 다음 문제로 넘어간다.
- 시작 조건: `학습 진행` 화면에 있다.
- 기본 흐름:
  1. 사용자가 단답형이면 답을 입력하고 제출한다.
  2. 객관식이면 보기 중 하나를 선택한다.
  3. 맞으면 큰 파란 `O`를 보여준다.
  4. 틀리면 정답을 중앙에 크게 보여준다.
  5. 설정된 지연시간 후 다음 문제로 넘어간다.

## UC-08 학습 완료 후 전체 다시 학습

- 사용자 목표: 방금 학습한 같은 범위를 처음부터 다시 푼다.
- 시작 조건: 한 세션이 끝났다.
- 기본 흐름:
  1. 앱이 완료 요약과 함께 `전체 다시 학습` 버튼을 보여준다.
  2. 사용자가 버튼을 누른다.
  3. 앱이 같은 범위 카드를 다시 섞어서 새 세션을 시작한다.

## UC-09 학습 완료 후 오답만 복습

- 사용자 목표: 틀린 카드만 다시 학습한다.
- 시작 조건: 한 세션이 끝났고 오답 카드가 하나 이상 있다.
- 기본 흐름:
  1. 앱이 완료 화면에 `오답만 학습` 버튼을 보여준다.
  2. 사용자가 버튼을 누른다.
  3. 앱이 틀렸던 카드만 모아 새 세션을 시작한다.
- 예외 흐름:
  - 오답이 하나도 없으면 버튼을 비활성화하고 `오답 없음`으로 표시한다.

## UC-10 학습 후 다른 작업으로 전환

- 사용자 목표: 학습 후 설정이나 탐색으로 되돌아간다.
- 시작 조건: 덱 상세, Day 상세, 학습 준비, 학습 진행 중 하나의 화면에 있다.
- 기본 흐름:
  1. 사용자가 상단의 이동 버튼을 사용한다.
  2. 앱이 메인 목록, 덱 상세, 학습 준비 등 적절한 이전 흐름으로 이동시킨다.

## UC-11 이미지가 포함된 카드 작성

- 사용자 목표: 텍스트 대신 이미지가 포함된 문제나 정답을 만든다.
- 시작 조건: `카드 편집` 화면에 있다.
- 기본 흐름:
  1. 사용자가 문제 또는 정답 편집 영역을 선택한다.
  2. 사용자가 이미지 복사 후 `Ctrl+V`로 붙여넣는다.
  3. 앱이 붙여넣은 이미지를 카드 HTML에 삽입한다.
  4. 사용자가 `카드 저장`을 누른다.
  5. 앱이 이미지가 포함된 카드를 저장하고 미리보기에도 즉시 반영한다.

## UC-12 잘못된 카드 저장 방지

- 사용자 목표: 문제나 정답이 비어 있는 카드를 실수로 저장하지 않는다.
- 시작 조건: `카드 편집` 화면에 있다.
- 기본 흐름:
  1. 사용자가 문제 또는 정답 중 하나만 입력한다.
  2. 사용자가 저장 버튼을 누른다.
  3. 앱은 내용이 부족한 상태면 저장하지 않는다.
- 기대 결과:
  - 불완전한 카드가 저장소에 들어가지 않는다.

## UC-13 카드 삭제 후 Day 재구성

- 사용자 목표: 필요 없는 카드를 제거하고 Day 구조를 유지한다.
- 시작 조건: 저장된 카드가 하나 이상 있다.
- 기본 흐름:
  1. 사용자가 카드 목록에서 대상 카드를 선택한다.
  2. 사용자가 `카드 삭제`를 누른다.
  3. 앱이 카드를 제거한다.
  4. 앱이 남은 카드 수 기준으로 Day 구조를 다시 계산한다.

## UC-14 대량 입력은 필요할 때만 사용

- 사용자 목표: 카드 한 장씩 편집에 집중하다가, 필요할 때만 여러 장을 한 번에 넣는다.
- 시작 조건: `카드 편집` 화면에 있다.
- 기본 흐름:
  1. 기본 상태에서는 대량 입력 영역이 접혀 있다.
  2. 사용자가 `대량 입력 열기`를 누른다.
  3. 앱이 대량 입력 영역을 펼친다.
  4. 사용자가 입력 후 가져오기를 수행한다.
  5. 사용자가 `대량 입력 숨기기`를 눌러 다시 주 편집 흐름으로 돌아간다.

## UC-15 세부 정보는 필요할 때만 펼치기

- 사용자 목표: 학습 준비 화면에서 핵심 결정만 빠르게 끝내고, 필요하면 세부 정보를 본다.
- 시작 조건: `학습 준비` 화면에 있다.
- 기본 흐름:
  1. 사용자가 학습 범위와 문제 유형만 먼저 고른다.
  2. 사용자가 `세부 정보 보기`를 누르면 앱이 추가 정보를 보여준다.
  3. 사용자가 `세부 정보 숨기기`를 누르면 다시 간결한 준비 화면으로 돌아간다.

## UC-16 앱 재실행 후 최근 상태 복원

- 사용자 목표: 앱을 다시 열었을 때 저장된 덱과 설정을 그대로 이어서 사용한다.
- 시작 조건: 이전에 앱에서 덱과 설정을 저장한 적이 있다.
- 기본 흐름:
  1. 사용자가 앱을 종료한다.
  2. 다시 앱을 실행한다.
  3. 앱이 자동 저장된 JSON 상태를 읽는다.
  4. 메인 `덱 목록` 화면에서 저장된 덱과 통계가 그대로 보인다.

## UC-17 Day 범위만 골라 학습

- 사용자 목표: 전체 덱이 아니라 특정 Day만 집중 복습한다.
- 시작 조건: Day가 생성된 덱이 있다.
- 기본 흐름:
  1. 사용자가 덱 상세 또는 Day 상세에서 학습 준비로 이동한다.
  2. 학습 범위를 특정 Day로 지정한다.
  3. 앱이 그 Day에 속한 카드만 대상으로 세션을 만든다.
  4. 학습 완료 후 `전체 다시 학습`을 누르면 그 Day 범위만 다시 시작한다.

## UC-18 전체 덱 오답만 재학습

- 사용자 목표: 전체 범위를 한 번 돌린 뒤 틀린 카드만 다시 본다.
- 시작 조건: 전체 카드 범위로 세션을 끝냈고 오답이 하나 이상 있다.
- 기본 흐름:
  1. 앱이 완료 화면에 `오답만 학습` 버튼을 활성화해 보여준다.
  2. 사용자가 버튼을 누른다.
  3. 앱이 방금 틀린 카드만 새 세션으로 구성한다.
  4. 사용자는 불필요한 정답 카드를 다시 보지 않는다.

## UC-19 메인 화면에서 빠른 스캔

- 사용자 목표: 많은 덱 중 원하는 덱을 빠르게 찾는다.
- 시작 조건: 덱이 여러 개 있다.
- 기본 흐름:
  1. 사용자가 3열 기준의 덱 카드 그리드를 위에서 아래로 훑는다.
  2. 앱이 덱 이름, 요약, 대표 이미지를 일정한 위치에 보여준다.
  3. 사용자가 카드 우상단 `...` 메뉴로 편집 또는 삭제를 선택할 수 있다.
  4. 사용자가 원하는 덱 카드를 누르면 상세 화면으로 이동한다.

## UC-20 완료 직후 다음 행동 결정

- 사용자 목표: 학습이 끝난 직후 무엇을 해야 할지 고민하지 않는다.
- 시작 조건: 학습 세션이 종료되었다.
- 기본 흐름:
  1. 앱이 정답 수, 오답 수, 정답률을 먼저 보여준다.
  2. 핵심 선택지는 `오답만 학습`과 `전체 다시 학습` 두 개만 강조된다.
  3. 사용자가 곧바로 다음 학습 행동을 선택한다.

## UC-21 단축키로 새 덱 생성

- 사용자 목표: 마우스를 크게 움직이지 않고 새 덱 생성 화면으로 바로 간다.
- 시작 조건: 메인 `덱 목록` 화면에 있다.
- 기본 흐름:
  1. 사용자가 `Ctrl+N`을 누른다.
  2. 앱이 `덱 이름 정하기` 화면으로 이동한다.
  3. 사용자가 즉시 이름 입력을 시작한다.

## UC-22 단축키로 카드 저장

- 사용자 목표: 카드 편집 중 저장을 더 빠르게 반복한다.
- 시작 조건: `카드 편집` 화면에서 문제와 정답을 편집 중이다.
- 기본 흐름:
  1. 사용자가 문제와 정답을 입력한다.
  2. 사용자가 `Ctrl+S`를 누른다.
  3. 앱이 현재 카드를 저장한다.
  4. Day 구조가 자동으로 다시 계산된다.

## UC-23 단축키로 보조 기능 열기

- 사용자 목표: 대량 입력 같은 보조 기능을 필요할 때만 빠르게 연다.
- 시작 조건: `카드 편집` 화면에 있다.
- 기본 흐름:
  1. 사용자가 `Ctrl+Shift+I`를 누른다.
  2. 앱이 대량 입력 영역을 펼치거나 숨긴다.
  3. 사용자는 메인 편집 흐름을 방해받지 않고 보조 작업에 들어간다.

## UC-24 숫자 키로 객관식 답안 제출

- 사용자 목표: 객관식 문제를 마우스 없이 빠르게 푼다.
- 시작 조건: `학습 진행` 화면에서 객관식 문제가 표시되어 있다.
- 기본 흐름:
  1. 앱이 각 보기에 `1`, `2`, `3`, `4` 힌트를 함께 보여준다.
  2. 사용자가 해당 숫자 키를 누른다.
  3. 앱이 그 보기를 선택한 것처럼 채점한다.

## UC-25 완료 화면 단축키로 재학습 선택

- 사용자 목표: 완료 직후 다음 행동을 키보드로 바로 고른다.
- 시작 조건: 학습이 끝난 상태다.
- 기본 흐름:
  1. 앱이 `오답만 학습`에 `1`, `전체 다시 학습`에 `2` 힌트를 보여준다.
  2. 사용자가 `1` 또는 `2`를 누른다.
  3. 앱이 해당 재학습 흐름을 시작한다.

## UC-26 3번 이내로 전체 학습 시작

- 사용자 목표: 앱을 켠 뒤 너무 깊게 들어가지 않고 곧바로 학습을 시작한다.
- 시작 조건: 메인 `덱 목록` 화면에서 원하는 덱을 보고 있다.
- 기본 흐름:
  1. 사용자가 덱 카드를 누른다.
  2. 사용자가 `전체 학습`을 누른다.
  3. 사용자가 `학습 시작`을 누른다.
- 기대 결과:
  - 자주 쓰는 전체 학습 경로는 3번 이내 상호작용으로 시작된다.

## UC-27 자주 쓰는 상위 작업만 빠르게 노출

- 사용자 목표: 자주 하는 행동을 가장 먼저 찾는다.
- 시작 조건: 메인 `덱 목록` 화면에서 덱 카드가 보인다.
- 기본 흐름:
  1. 사용자는 먼저 덱 카드 자체를 보고 원하는 덱을 고른다.
  2. 세부 구조와 학습, 카드 편집은 덱 상세 화면에서 찾는다.
- 기대 결과:
  - 학습과 편집 같은 상위 빈도 행동이 세부 관리보다 먼저 노출된다.
