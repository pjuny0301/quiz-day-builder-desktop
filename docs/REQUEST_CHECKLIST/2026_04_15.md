# 2026-04-15 요청 반영 체크리스트

이 문서는 2026-04-15에 추가된 사용자 요청을 **원문에 가깝게 유지**하면서, 각 항목의 시행 여부를 표시한다.

## 기본 원칙

- 카드 편집 창은 **단일 카드 편집 창**으로 유지한다.
- 대량입력은 **별도 전용 창**으로 분리한다.
- 이 문서의 각 항목은 구현 후 `시행` 또는 `미시행`으로 표시한다.

## 요청 목록 및 시행 여부

### 1. 카드내용 편집 창

#### 1-1 수정

- [시행] 1-1-1 문제 작성 위젯과 정답 작성 위젯은 같은 작성 축에 둔다.
- [시행] 1-1-1 미리보기 위젯들은 별도 미리보기 축에 두고, 작성 축과 나란히 배치한다.
- [시행] 1-2-1 `카드 수정 저장` 문구를 `카드 저장`으로 바꾼다.
- [시행] 1-2-2 대량입력은 따로 창을 띄우고, 그 창에서도 이미지 복붙이 가능하게 한다.
- [시행] 1-2-3 대량입력의 각 문제별 구분은 블로그 스타일의 구분선 디자인으로 한다.
- [시행] 1-2-4 대량입력은 텍스트 문제/정답으로 따로 받을 수 있게 하고, 입력된 쌍은 자동으로 구분선이 있는 카드 블록으로 내려보낸다.

#### 1-2 추가

- [시행] 1-2-1 새 카드 단축키를 추가한다.
  적용: `Ctrl+N`

### 2. 덱 상세 창

#### 2-1 수정

- [시행] 2-1-1 카드가 있는 덱에서 `전체 학습` 또는 `학습` -> `단답형/선택형/혼합형` -> `학습 시작` 시 하얀 창만 뜨는 버그를 수정한다.
- [시행] 2-1-2 `Day 리스트` 텍스트 위젯과 덱 제목 위젯을 통합한다.
  적용: 별도 `Day 리스트` 헤더를 두지 않고 덱 제목 영역으로 흡수.
- [시행] 2-1-3 Day의 `열기` 버튼 위 텍스트를 `누적 O/X` 형식에서 `총 n1문제 중 n2문제 맞춤` 형식으로 바꾼다.

### 3. 새 덱 버튼 -> 덱 이름 정하기 창

#### 3-1 추가

- [시행] 3-1-1 `Day당 카드 수`와 `자동분할 미리보기` 위젯을 제외하고 덱 편집과 같은 구성으로 만든다.
  적용: 덱 이름, 대표 이미지, 정답 표시 지연시간을 공통 입력 구성으로 맞춤.
