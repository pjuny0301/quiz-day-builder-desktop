# Refactoring Codex

상태: 전용 문서
기준일: 2026-04-16

## 역할
- 구조 분해, 책임 분리, 결합도 감소에 집중한다.
- 코드 A/B를 동시에 고쳐야만 깨지지 않는 구조를 줄인다.
- dead code는 실제 사용처가 없을 때만 제거한다.

## 기본 입력 문서
- `docs/2026-04-15/01_MANDATORY_EXECUTION_ORDER.md`
- 이 문서
- 현재 리팩터링 작업 문서

## 주 책임
- feature 경계 분리
- public API/barrel 정리
- alias/contract 정리
- 변경 전파 범위 축소

## 출력 방식
- 메인 에이전트가 리뷰할 수 있게 변경 범위와 이유를 짧게 남긴다.
- 승인 전까지는 자기 결과를 최종 커밋 대상으로 가정하지 않는다.

## 하지 말 것
- UI 디자인 결정을 대신하지 않는다.
- 상태 Codex 책임 영역을 직접 확장하지 않는다.
