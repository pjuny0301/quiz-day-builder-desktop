# Responsibility Refactor

상태: 시행 중
기준일: 2026-04-16

## 이번 리팩터링 목표
- 화면 파일이 상태 전이, 디버그 시나리오, 자동화 등록까지 전부 떠안지 않게 분리한다.
- 학습 세션은 `화면 렌더링`, `세션 제어`, `디버그/자동화 유틸`로 나눈다.
- 실제 사용처가 없는 코드만 같이 제거한다.

## 이번 적용 범위
- `StudySessionWindow.tsx`
- `src/features/study/session/*`
- 사용처 없는 컴포넌트/유틸 정리

## 이번 패스에서 하지 않는 것
- UI 디자인 전면 수정
- 저장 구조 변경
- 카드 편집/덱 편집 화면의 대규모 이동

## 역할 분리 기준
- `windows`: 화면 진입점
- `features/study/session`: 학습 세션 전이와 디버그 시나리오
- `components`: 여러 화면이 같이 쓰는 공용 UI
- `domain/store`: 화면 바깥의 순수 로직과 저장 계층
