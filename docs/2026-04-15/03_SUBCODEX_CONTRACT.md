# Implementation Sub-Codex Contract

상태: 강제 적용
기준일: 2026-04-15

## 구현 담당 하위 Codex 작업 방식
- 상태관리와 퀴즈로직만 맡는다.
- 화면 디자인과 배치를 바꾸지 않는다.
- 메인 에이전트가 정의한 공개 계약을 유지하거나, 바꿔야 하면 변경 이유를 함께 남긴다.
- 구현 담당 하위 Codex의 본업은 구현이다.
- 구현 담당 하위 Codex의 본업은 커밋 심사가 아니다.

## 구현 단계
1. 상태/로직 중심 변경 수행
2. 타입 및 최소 빌드 기준 확인
3. 변경 파일과 공개 엔트리 포인트를 정리
4. 메인 에이전트가 나중에 읽을 수 있게 요약 남김

## 출력 형식
- 변경 목적
- 변경 파일 목록
- 공개 타입/함수 변경점
- 메인 에이전트가 연결할 포인트
- 남은 위험

## 토큰 절감 원칙
- 구현 담당 하위 Codex 메시지는 바로바로 읽지 않는다.
- 커밋 단위 심사 때만 읽는다.
- 메인 에이전트가 재작업 지시를 줄 때는 다시 구현 가능한 정도로 구체적이어야 한다.
