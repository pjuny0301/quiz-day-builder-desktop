# Frontend vs State/Quiz Logic Split

상태: 강제 적용
기준일: 2026-04-15

## 목적
- 메인 에이전트는 프론트엔드 중심으로 움직인다.
- 구현 담당 하위 Codex는 상태관리와 퀴즈로직 중심으로 움직인다.
- 심사는 구현 담당 하위 Codex의 역할이 아니다.
- 두 영역이 같은 파일을 동시에 건드리지 않도록 책임 경계를 명시한다.

## 메인 에이전트 책임
- 화면 구조
- 사용자 흐름
- UI 컴포넌트
- 스타일링
- 화면별 문구
- 디자인 일관성
- 프론트엔드 계약 정의
- 커밋 심사 수행 또는 리뷰 단계 주도

## 구현 담당 하위 Codex 책임
- 앱 상태 모델 정규화
- 퀴즈 세션 상태 전이
- 채점 규칙
- 보기 생성 규칙
- Day 분할 규칙
- 저장 구조와 저장 어댑터
- 상태 계층 리팩터링

## 메인 에이전트 소유 파일군
- `src/windows/**`
- `src/components/**`
- `src/styles.css`
- `src/App.tsx`
- `src/main.tsx`
- 문서 폴더

## 구현 담당 하위 Codex 소유 파일군
- `src/state/**`
- `src/lib/state-utils.ts`
- `src/lib/types.ts`
- 앞으로 분리될 `src/domain/**`
- 앞으로 분리될 `src/store/**`
- 저장 구조 관련 Tauri 연동 보조 계층

## 공유 경계
- 화면은 도메인 모듈의 공개 함수와 타입만 사용한다.
- 화면에서 퀴즈 규칙을 직접 계산하지 않는다.
- 도메인 모듈은 React 화면 구현에 의존하지 않는다.

## 금지 사항
- 메인 에이전트가 상태 전이 로직을 화면 파일 안에 새로 넣는 것
- 구현 담당 하위 Codex가 화면 레이아웃과 스타일을 임의로 바꾸는 것
- 구현 담당 하위 Codex가 심사 전담처럼 행동하는 것
- 서로 같은 파일을 동시 편집하는 것
