# Commit Review Protocol

상태: 강제 적용
기준일: 2026-04-15

## 목적
- 구현 담당 하위 Codex와 심사 단계를 분리한다.
- 심사는 별도 단계이며, 구현 역할과 섞지 않는다.

## 심사 주체
- 기본 심사 주체는 메인 에이전트다.
- 필요하면 별도 리뷰 하위 Codex를 둘 수 있다.
- 구현 담당 하위 Codex를 심사 전담으로 간주하지 않는다.

## 심사 루프
1. 커밋 생성
2. 커밋 심사
3. 승인 또는 거부
4. 거부 시 구체 사유 기록
5. 구현 담당 하위 Codex가 수정
6. 재심사

## 거부 형식
- 거부 대상 커밋
- 핵심 문제
- 왜 문제인지
- 수정해야 할 파일
- 수정해야 할 함수 또는 타입
- 기대 결과

## 운영 원칙
- 심사와 구현은 역할을 섞지 않는다.
- 하위 Codex 응답은 심사 시점에 묶어서 읽는다.
- 거부 사유는 즉시 재작업 가능할 만큼 구체적이어야 한다.
