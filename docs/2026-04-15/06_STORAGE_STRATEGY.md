# Storage Strategy

상태: 시행
기준일: 2026-04-15
적용 단계: 7. 퀴즈 저장 방식 정비

## 목적
- 저장 파일 손상 때문에 앱 전체가 빈 화면이나 로딩 실패 상태로 무너지지 않게 한다.
- 데스크톱 빌더와 이후 모바일 퀴즈앱이 같은 저장 문서 구조를 공유할 수 있게 한다.
- 저장 동작은 단순 JSON을 유지하되, 쓰기 실패와 부분 저장에 더 강하게 만든다.

## 현재 선택
- 주 저장 파일: `app_data.json`
- 백업 파일: `app_data.backup.json`
- 임시 파일: `app_data.json.tmp`
- 문서 메타: `meta.schema_version`

## 시행 내용
- 저장 시 기존 주 파일을 먼저 백업 파일로 복사한다.
- 새 상태는 임시 파일에 먼저 쓴다.
- 임시 파일 쓰기가 끝난 뒤 주 파일을 교체한다.
- 앱 시작 시 주 파일이 손상되었으면 백업 파일을 읽어 복구한다.
- 저장 문서에는 `schema_version`을 붙여 이후 마이그레이션 기준점으로 쓴다.

## 이유
- 기존 방식은 `fs::write(app_data.json)` 한 번으로 끝나서 저장 중단이나 비정상 종료 때 파일 손상에 취약했다.
- 현재 프로젝트는 안드로이드용 퀴즈앱까지 고려하므로 저장 형식은 단순해야 하지만, 복구 가능성은 높여야 한다.
- JSON 구조를 유지하면 프론트엔드와 도메인 정규화 로직을 다시 크게 바꾸지 않아도 된다.

## 이후 기준
- 저장 구조 교체가 필요하면 이 문서 아래에 새 단계 문서를 추가하고 교체 이유를 적는다.
- 모바일 저장 백엔드를 별도로 붙이더라도 문서 스키마는 최대한 동일하게 유지한다.
