# Mandatory Read Before Coding

이 문서는 이 프로젝트를 수정하기 전에 **반드시 먼저 읽어야 하는 규칙**을 담는다.  
새 기능 추가, UI 수정, 리팩터링, 버그 수정 모두 예외 없이 이 문서를 선행 기준으로 삼는다.

## Must Read Order

1. 이 문서
2. `README.md`
3. `docs/CONFIRMED_DIRECTION.md`
4. `docs/INFERRED_INTENT.md`
5. `docs/USE_CASES.md`
6. `docs/UX_LAWS.md`
7. `docs/NIELSEN_HEURISTICS.md`

## Non-Negotiable Rules

- 구현 전에 위 문서들을 먼저 읽고 현재 사용자 방향을 다시 확인한다.
- 함수마다 그 함수의 목적을 설명하는 **짧은 한 줄 주석**을 바로 위에 단다.
- 급한 땜질식 수정으로 끝내지 말고, **확장성 있게 리팩터링**하면서 구조를 정리한다.
- 화면 하나에는 핵심 역할 하나만 강하게 두고, 값 설정은 가능한 한 **전용 화면**에서 처리한다.
- 가능하면 새 OS 창을 계속 띄우기보다, **단일 메인 창 안에서 역할별 화면 전환**으로 체감 속도를 유지한다.
- 요구사항이 바뀌면 코드보다 먼저 문서 기준을 갱신하고, 그 기준에 맞춰 구현한다.

## Refactoring Standard

- 상태, 저장, 창 라우팅, 재사용 UI는 서로 분리된 책임으로 유지한다.
- 새 기능이 들어와도 단일 파일이 비대해지지 않게 분리한다.
- 현재 기능만 돌아가게 만드는 코드보다, 다음 요구를 붙이기 쉬운 구조를 우선한다.

## Comment Standard

- export 함수뿐 아니라 내부 헬퍼 함수에도 목적 주석을 단다.
- 주석은 구현 설명이 아니라 **존재 이유와 역할**을 짧게 적는다.
- 주석이 없는 새 함수는 미완성으로 간주한다.
